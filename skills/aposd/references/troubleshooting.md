# Troubleshooting

| Problem | Cause | Fix |
|---------|-------|-----|
| Skill doesn't trigger on refactoring | Description says "writing, reviewing, or modifying any code" | Verify the agent reads the skill description. Refactoring is "modifying" — trigger is correct. |
| Rules conflict (e.g., General-Purpose vs Simplicity) | Principles have tension by design | Use the complexity lens: the option that better reduces cognitive load wins. |
| Agent uses APOSD vocabulary without design depth | Vocabulary is not investment (Rule 1 Red Flag) | Check if the interface actually changed. If not, it's tactical regardless of language. |
| User explicitly rejects strategic approach | Time pressure or "just make it work" | State the tradeoff, document it, then proceed tactically. Routing Rule 3 covers this. |
